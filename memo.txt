global feature 에서 feature 을 놓치게 되어도 skip connection 으로 잡을 가능성의 여지가 있다.
그러나 만약 local feature 에서도 놓치게 되었다면?
즉,
1. global feature 자체에서도 정보를 얻는 것을 실패하였고,
2. local feature 에서도 global 정보가 필요한 상황이므로 놓치게 될 수 있다.
이러한 문제를 해결하기 위해, global feature 를 local feature 에서도 사용할 수 있도록 하는 방법이 필요하다.

1) global feature extracting
2) when extracting local feature, use global feature

현재의 algorithm 은 feature을 뽑고 그 후에 합해지게 된다.
그렇게 하지 말고, feature 을 연속적으로 뽑을 수 있도록 한다.
